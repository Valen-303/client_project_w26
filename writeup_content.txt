问题 1: How you iterated on Deliverable #2 feedback
=====================================================

Two pieces of feedback were addressed:

1. Footer needed more useful info. I added the CDO's physical address (105 S State St, Ann Arbor), a mailto email link, and social media links (LinkedIn, X/Twitter). The footer-links grid now has 4 items instead of 2, giving students more ways to connect with UMSI.

2. FAQs could be accordions. I converted the FAQ sections on the Resume and Portfolio pages from static lists into interactive accordions using HTML5 <details>/<summary> elements. Each question is clickable — it expands to show the answer, and collapses when clicked again. This keeps the pages cleaner and lets students find answers faster. The accordions have visible focus and hover states for accessibility.

=====================================================
问题 2: How your design optimizes for larger screens
=====================================================

I used a min-width: 768px media query. Key layout changes at this breakpoint:

- Header: flex layout, logo and title side by side
- Nav: 2x2 grid → single row of 4 links
- Card grids / Quick links / Resume resources: 2 columns → 4 columns
- FAQ accordions and Resource lists: become 2-column grids
- Main content: capped at 960px width, centered
- Images: max-width 600px, centered
- Video iframe: 200px → 400px tall
- Footer: footer-links become a 4-column grid
- Back to Top button: becomes fixed in bottom-right corner

Every page is affected by these changes.

=====================================================
问题 3: How updates fit UM Branding Guidelines
=====================================================

Same UM brand colors throughout — Michigan Blue (#00274C), Maize (#FFCB05), and supporting palette. The wider layout gives the design more room to breathe without changing the color scheme or typography. The horizontal header with the UMSI logo looks more institutional on a wide screen. The 4-column grids feel polished and organized. The enhanced footer with contact info and social links mirrors what you'd see on UM's official sites.

=====================================================
问题 4: Unique/challenging contribution
=====================================================

I added a "Back to Top" button on every page. On mobile, it's a full-width link at the bottom of the content. On desktop, it's fixed in the bottom-right corner so it's always accessible while scrolling. It links to id="top" on the body element and is styled with UM colors, hover effects, and a visible focus outline for keyboard users. This helps students navigate long content pages (like the Resume page) without having to scroll all the way back up to the nav.

=====================================================
问题 5: How you ensured accessibility
=====================================================

All Deliverable #2 accessibility features carry over: skip-to-main link, descriptive alt text, WCAG AA color contrast, multi-indicator active nav (color + border + underline), visible focus outlines, and prefers-reduced-motion media query.

New for Deliverable #3: the FAQ accordions use native HTML5 <details>/<summary>, which are keyboard-accessible by default — users can Tab to each question and press Enter/Space to toggle it. I added custom focus (3px yellow outline) and hover (gray background) states to the summary elements. The Back to Top button also has proper focus and hover states. The 404 page includes the same nav, skip link, and footer as all other pages. Validated all 5 pages with WAVE, aXe, W3C HTML, and W3C CSS — no errors.

=====================================================
问题 6: How you designed the 404 page
=====================================================

The 404 page uses the same header, nav, and footer as the rest of the site so users know they're still on the UMSI Career Hub. The main area shows a big "404" in Maize, a friendly message explaining the page wasn't found, and the same quick-link button grid from the Home page (Home, Resumes, Networking, Portfolio) so users can get back on track immediately. The file is named 404.html — GitHub Pages automatically serves it for nonexistent URLs.
